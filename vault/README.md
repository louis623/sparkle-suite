# Neon Rabbit Memory Vault

This is the AI-readable memory vault for the Neon Rabbit / Sparkle Suite project.

**All files are plain Markdown.** Any AI tool (Claude, Gemini, Co-work, or any other agent) should read the full vault at the start of every session before doing any work. This vault is the single source of truth for project context, decisions, client status, and open items.

## How to Use This Vault

1. At session start, read every file in this folder top to bottom.
2. Do not assume you already know the context — always re-read.
3. After significant work sessions, update the relevant files to keep the vault current.
4. Log significant sessions in `session-log.md`.

---

## File Index

| File | Contents |
|------|----------|
| `README.md` | This file. What the vault is, how to use it, file index. |
| `project-state.md` | Current project phase, platform status, priorities, and memory architecture overview. |
| `clients.md` | Full client roster with status — live, under construction, and waitlist. |
| `decisions.md` | Every key architectural, tooling, and operational decision, logged with date and rationale. |
| `stack.md` | Confirmed tech stack and full tool roster across infrastructure, automation, memory, comms, payments, and security. |
| `open-items.md` | Everything deferred, undecided, or waiting — tasks, planning sessions, and open questions. |
| `session-log.md` | Running log of significant work sessions with topics covered. |
